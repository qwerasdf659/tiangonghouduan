> 版本：v1.0 | 日期：2026-07-15 | 面向：运营人员
> 数据来源：真实数据库配置（restaurant_points_dev）+ 实际发放代码，非估算。

## 二、核心计算公式

消费审核通过时，一笔消费会触发**三条独立的发放**：用户可见积分、预算账户、星石配额。

### 关键概念：总倍数（加法叠加）

```
总倍数 = 1（基础打底） + 等级加成率 + 活动加成率
         最终不超过 3.0（硬封顶）
```

- **等级加成率**：由用户当前成长等级决定（0% ~ 50%）。
- **活动加成率**：由命中的消费加成活动决定（0 = 无活动；可配 0 ~ 2.0）。
- 两者**相加**，不是相乘。

### ① 用户可见积分（用户能看能花）

```
基础分     = 消费金额 × 1（基础比例）
可见积分   = 基础分 × 总倍数
           = 消费金额 × (1 + 等级加成率 + 活动加成率)
```

### ② 预算账户（内部奖池弹药，用户不可见）

```
预算注入 = min(
             消费金额 × 0.08（预算比例） × 总倍数,
             消费金额 × 16%（注入率硬封顶）
           )
```
> 预算账户同样被等级 + 活动倍率放大，但多一道 16% 封顶。预算越厚，抽奖越容易出高档奖品。

### ③ 星石配额（既有链路，不受倍率影响）

```
星石配额 = 消费金额 × 1（配额比例，向下取整）
```
> 星石配额**不受**等级/活动倍率影响，按固定比例发放。

## 三、实例速查表（消费 100 元）

### 3.1 只有等级、无活动（活动加成率 = 0）

| 等级 | 总倍数 | 可见积分 | 预算注入 |
|---|---|---|---|
| 铜卡 | 1.00 | 100 | 8 |
| 银卡 | 1.05 | 105 | 8.4→8 |
| 金卡 | 1.10 | 110 | 8.8→9 |
| 黑金卡 | 1.25 | 125 | 10 |
| 荣耀殿堂 | 1.50 | 150 | 12 |

### 3.2 叠加活动加成（活动加成率 = 0.5，即多送50%）

| 等级 | 总倍数 | 可见积分 | 预算注入 |
|---|---|---|---|
| 铜卡 | 1 + 0 + 0.5 = 1.50 | 150 | 100×0.08×1.5 = 12 |
| 黑金卡 | 1 + 0.25 + 0.5 = 1.75 | 175 | 100×0.08×1.75 = 14 |
| 荣耀殿堂 | 1 + 0.5 + 0.5 = 2.00 | 200 | 100×0.08×2.0 = 16（刚好触顶） |

### 3.3 触顶场景（活动加成率配得过高）

| 场景 | 理论总倍数 | 实际总倍数 | 可见积分 | 说明 |
|---|---|---|---|---|
| 铜卡 + 活动2.0 | 1+0+2.0 = 3.0 | 3.0 | 300 | 正好触顶 |
| 荣耀殿堂 + 活动2.0 | 1+0.5+2.0 = 3.5 | **3.0** | 300 | 活动加成被截断到 1.5（等级优先） |

> ⚠️ 注意：活动加成率配到 2.0 时，高等级用户的活动加成会被 3.0 封顶"吃掉"，
> 铜卡和荣耀殿堂拿的一样多（都 300），失去等级差异化意义。

## 四、配置活动加成率的建议

### 取值范围
- `bonus_rate`（活动加成率）：**0 ~ 2.0**（0=无活动，0.5=多送50%，1.0=翻倍，2.0=多送200%）。
- 后端会拦截超上限的配置。

### 运营建议值
- **推荐区间：0.1 ~ 1.0**（多送 10% ~ 100%）。
- 不建议配到 2.0：会和高等级的等级加成挤占 3.0 封顶，导致高低等级用户拿一样多，浪费等级体系。
- 想让高等级用户在活动期间也有明显优势，活动率建议 ≤ 1.5，给等级留出叠加空间。

### 活动范围（全平台 vs 商家专属）
- **全平台活动**：门店/商家 ID 都留空 → 所有商家所有用户都享受。
- **商家专属活动**：填门店/商家 ID → 只在该商家/门店消费享受。
- **商家专属优先**：同一笔消费同时命中平台活动和商家专属活动时，用商家专属的。

### 生效时机
- 活动加成率在**用户提交小票时锁定**，随消费记录落表。
- 活动中途启停/调率，**不影响已提交的小票**（用户承诺一致）。

## 五、几条重要规则（避免踩坑）

1. **活动无配置时加成率 = 0，不是 1.0**：0 表示"不额外加"，用户照常拿基础分×等级倍率，不会没积分。
2. **总倍数永远 ≥ 1.0**：那个打底的 1 保证用户至少拿到"基础分 × 等级倍率"，活动为 0 只是没有额外加成。
3. **加法不是乘法**：等级和活动是"加成率相加"，不是"倍率相乘"。黑金卡(+25%) + 活动(+50%) = 总 +75%（1.75倍），不是 1.25×1.5。
4. **预算账户有 16% 封顶，可见积分没有**：高倍率时预算会被 16% 截断，可见积分只受 3.0 封顶。两者高倍率时会脱钩。
5. **星石配额不受倍率影响**：固定按消费额×1 发放，活动/等级都不放大它。

## 六、数据保密红线（务必遵守）

| 数据 | 能否对用户展示 |
|---|---|
| 等级倍率、活动加成率、活动展示名 | ✅ 可展示（营销激励） |
| **预算账户注入、预算比例 0.08、16% 封顶** | ❌ **禁止**（商业机密，决定抽奖出什么奖品） |
| 星石配额内部机制 | ❌ 禁止对外 |

> 小程序只会显示"消费多送X%积分"这类营销信息，绝不显示预算账户如何被放大——
> 这是防止用户算计抽奖系统、防竞品抄袭的核心红线。运营对外宣传也须遵守。
