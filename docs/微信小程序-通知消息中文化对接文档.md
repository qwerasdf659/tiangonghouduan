# 微信小程序前端 - 通知消息中文化修复对接文档

## 修复概述

**问题**：微信小程序"消息通知"页面显示英文资产代码（如 `red_core_shard`），用户无法理解。

**根因**：后端 `NotificationService` 在生成通知消息时，直接将 `asset_code`（英文标识符）拼入 `content` 字段，未转换为中文 `display_name`。

**修复**：后端已完成修复，通知 `content` 字段现在直接包含中文名称。前端无需做任何映射或翻译逻辑。

---

## 修复后的通知数据结构

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v4/user/notifications` | 获取通知列表（分页） |
| GET | `/api/v4/user/notifications/unread-count` | 获取未读数量 |
| POST | `/api/v4/user/notifications/mark-read` | 批量标记已读 |
| POST | `/api/v4/user/notifications/:id/read` | 单条标记已读 |

### 请求头

```
Authorization: Bearer <access_token>
```

### GET /api/v4/user/notifications 查询参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | number | 1 | 页码 |
| page_size | number | 20 | 每页数量（最大50） |
| type | string | - | 按通知类型筛选 |
| is_read | string | - | 按已读状态筛选（0=未读，1=已读） |

## 前端展示说明

### content 字段直接展示

`content` 字段已经是完整的中文消息，前端直接渲染即可，**不需要做任何翻译或映射**。

修复前（错误）：`"您已成功购买 100 个 red_core_shard，支付 500 星石。"`
修复后（正确）：`"您已成功购买 100 个 红源晶碎片，支付 500 星石。"`

### metadata 字段用途

`metadata` 中保留了原始业务数据（如 `offer_asset_code`），供前端做跳转逻辑使用（如点击通知跳转到对应的交易详情页），**不用于展示**。

## WebSocket 实时推送

后端通过 Socket.IO 推送实时通知，事件名：`new_notification`

```javascript
// 微信小程序 Socket.IO 监听示例
socket.on('new_notification', (notification) => {
  // notification 结构与 API 返回的单条通知一致
  // content 已经是中文，直接展示
  console.log(notification.title)   // "🎉 购买成功"
  console.log(notification.content) // "您已成功购买 100 个 红源晶碎片，支付 500 星石。"
})
```
