# 奖品图片真机显示修复方案

**日期**：2026-03-09
**状态**：已修复并验证通过

---

## 一、问题描述

抽奖页面 12 个奖品图片在微信开发者工具中全部正常显示，但在真机上只有"积分 ×10"能显示，其余 11 个（钻石、红水晶碎片）显示空白。

## 二、排查过程

### 2.1 已排除的原因

| 排查项 | 结论 |
|---|---|
| 后端 API 数据 | ✅ 12 个奖品的 `image.url` 全部有值 |
| 前端字段读取 | ✅ 正确读取 `prize.image.url` |
| 域名白名单 | ✅ 积分图片同域名能加载，说明域名已配置 |
| 图片文件格式 | ✅ 三种图片均为 256×256 PNG，格式一致 |
| 微信缓存 | ✅ 删除小程序重进后仍然复现 |

### 2.2 定位到的根因

通过 `binderror` + `bindload` 调试日志确认：

```
[IMG-OK]   积分 ×10     | src= .../1771668349743_a6329ab6af399193.png  ← 200
[IMG-FAIL] 钻石 ×500    | src= .../1771668349696_975cb89a85cb8a1c.png  ← 404
[IMG-FAIL] 红水晶碎片 ×50 | src= .../1771668349848_7393439468341796.png  ← 404
```

**同一个 URL，浏览器/curl/wx.downloadFile 返回 200，但微信 `<image>` 组件真机返回 404。**

微信 `<image>` 组件在真机上有独立的图片下载通道（非标准 HTTP 请求），该通道对 Sealos 图片代理的部分 URL 行为异常。开发者工具不走这个通道，所以表现正常。

### 2.3 已尝试但失败的方案

| 方案 | 结果 | 失败原因 |
|---|---|---|
| 直接使用后端网络 URL | 部分失败 | `<image>` 组件真机对 DIAMOND/red_shard URL 返回 404 |
| 主包本地路径 `/images/icons/materials/*.webp` | 全部失败 | 分包无法引用主包静态资源（微信限制） |
| 分包本地路径 `/packageLottery/images/icons/materials/*.webp` | 全部失败 | `<image src="{{变量}}">` 数据绑定模式下真机不解析本地路径 |

## 三、最终解决方案

### 3.1 核心思路

不依赖 `<image>` 组件的内置下载机制，改用 `wx.downloadFile` 主动下载图片到本地临时路径，再将 `<image src>` 设置为本地临时文件路径。

### 3.2 实现流程

```
后端 API 返回 prize.image.url（网络 URL）
        ↓
addPrizeIcon() 赋值 prize_image_url = image.url
        ↓
setData() 渲染页面（此时 <image> 可能加载失败）
        ↓
downloadPrizeImages() 批量调用 wx.downloadFile
        ↓
下载成功 → setData 更新 prize_image_url = tempFilePath（本地临时路径）
        ↓
<image src> 从网络 URL 变为本地文件 → 真机正常渲染
```

### 3.3 关键代码

**`downloadPrizeImages` 函数**（`lottery-activity.ts`）：

```typescript
function downloadPrizeImages(prizes: any[], dataKey: string, ctx: any): void {
  prizes.forEach((prize: any, idx: number) => {
    const url = prize.prize_image_url
    if (!url || !url.startsWith('http')) {
      return
    }
    wx.downloadFile({
      url,
      success(res) {
        if (res.statusCode === 200 && res.tempFilePath) {
          ctx.setData({ [`${dataKey}[${idx}].prize_image_url`]: res.tempFilePath })
        }
      }
    })
  })
}
```

**调用位置**（`setData` 之后）：

```typescript
this.setData({ prizes, prizesForPreview, ... })

// 通过 wx.downloadFile 预下载奖品图片到本地临时路径
downloadPrizeImages(prizes, 'prizes', this)
downloadPrizeImages(prizesForPreview, 'prizesForPreview', this)
```

### 3.4 为什么 wx.downloadFile 能成功

| 对比项 | `<image>` 组件内置下载 | `wx.downloadFile` |
|---|---|---|
| 网络通道 | 微信私有图片通道 | 标准 HTTP 请求 |
| 请求头 | 微信自定义（可能含特殊 UA/Referer） | 标准微信 UA |
| 缓存机制 | 微信图片缓存层（可能缓存旧 404） | 独立下载，无图片缓存干扰 |
| 真机 DIAMOND URL | 404 | 200 |

## 五、注意事项

1. `wx.downloadFile` 需要域名在微信公众平台的 `downloadFile` 合法域名列表中（`omqktqrtntnn.sealosbja.site` 已配置）
2. 下载的临时文件在小程序本次生命周期内有效，退出后可能被清理
3. 12 个奖品图片实际只有 3 个不同的 URL（DIAMOND/red_shard/POINTS），`wx.downloadFile` 会对相同 URL 自动复用缓存
4. 调试代码（`console.log` / `onDebugImageLoad` / `onDebugImageError`）验证通过后需要清理
