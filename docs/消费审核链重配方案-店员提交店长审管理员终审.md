**审核人判定逻辑**（`ApprovalChainService._verifyOperatorPermission`）：满足任一即可审——
1. 角色池：你的角色 `role_id` == 节点 `assignee_role_id`；

> 叠加约束：非 admin 有门店/区域隔离（只能审本店）；`exclude_parties=1` 当事人回避（不能自审，super_admin lv≥110 豁免）；ops(lv30) 只读不能审。

**账号真实角色**

| 账号 | 角色 | role_level | 现状能审吗 |
| --- | --- | --- | --- |
| 13612227910（uid32 用户7910） | admin + super_admin | 100/110 | ✅ 能审全部（admin 兜底） |

```

  assignee_user_id= NULL

  exclude_parties = 1                       ← 店长不能审自己提交的单
```

| 账号 | 角色 | role_level |
| --- | --- | --- |
| 13612227910 | admin + super_admin | 100 / 110 |
| 13612227930 | user + merchant_manager | 0 / 40 |

「审核详情」按钮的显示由**微信小程序前端**自行判断（前端代码不在后端项目，后端不控制按钮显隐）。问题在于前端的显示条件**只覆盖了某一类角色**（如只认 `role_level≥100` 管理员，或只认某一种店长角色），没有把"管理员 + 店长"都纳入。
