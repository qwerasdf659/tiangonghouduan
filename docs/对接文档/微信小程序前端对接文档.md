# 天工平台 微信小程序前端对接文档

> **文档定位**：本文档面向微信小程序前端开发者，描述如何对接天工平台后端 API。
> **核对基准**：基于后端数据库项目当前真实代码（routes/v4/**、middleware/auth.js、utils/ApiResponse.js）实测整理，**非历史文档、非推测**。
> **权威源原则**：以**后端实际实现为唯一权威**。接口路径、字段名、响应格式一律以后端为准；前端**直接改用后端字段名，不做兼容映射层**。
> **整理日期**：2026年6月9日（北京时间）

## 一、基础约定

### 1.1 服务地址（Base URL）

| 环境 | Base URL |
|------|----------|
| 生产/外网 | `https://omqktqrtntnn.sealosbja.site` |
| WebSocket | `wss://omqktqrtntnn.sealosbja.site/ws` |

所有接口路径前缀：`/api/v4`。例如登录接口完整地址为 `https://omqktqrtntnn.sealosbja.site/api/v4/auth/login`。

> 实际域名以运维提供为准；小程序需在微信公众平台「开发管理 → 服务器域名」中配置 request 合法域名与 socket 合法域名。

### 1.2 请求头规范

| Header | 是否必填 | 说明 |
|--------|---------|------|
| `Content-Type` | 是（POST/PUT） | 固定 `application/json` |
| `Authorization` | 登录后必填 | 格式 `Bearer <access_token>` |
| `X-Device-Id` | 强烈建议 | 设备唯一标识，用于设备级多会话隔离；同设备重登替换旧会话，不同设备并存 |
| `Idempotency-Key` | 写操作必填 | 幂等键，详见 1.5。抽奖等写接口缺失会返回 400 |

### 1.3 时区

全链路北京时间（`Asia/Shanghai`, UTC+8）。所有时间戳形如 `2026-06-09T05:43:30.579+08:00`。

### 1.4 统一响应格式

后端所有业务响应 **HTTP 状态码恒为 200**（特殊认证失败除外），通过响应体的 `success` 字段判断业务成败。

成功响应：

```json
{
  "success": true,
  "code": "SUCCESS",
  "message": "操作成功",
  "data": { },
  "timestamp": "2026-06-09T05:43:30.579+08:00",
  "version": "v4.0",
  "request_id": "req_xxxxxxxx"
}
```

错误响应：

```json
{
  "success": false,
  "code": "INVALID_VERIFICATION_CODE",
  "message": "验证码错误或已过期",
  "data": {},
  "timestamp": "2026-06-09T05:43:30.579+08:00",
  "version": "v4.0",
  "request_id": "req_xxxxxxxx",
  "httpStatus": 401
}
```

分页响应（额外带 `pagination`）：

```json
{
  "success": true,
  "code": "PAGINATION_SUCCESS",
  "message": "查询成功",
  "data": [ ],
  "pagination": {
    "total": 100, "page": 1, "page_size": 20,
    "total_pages": 5, "has_next": true, "has_prev": false
  },
  "timestamp": "...", "version": "v4.0", "request_id": "..."
}
```

**前端处理约定**：
- 成功只读 `data`（分页同时读 `pagination`）。
- 失败读 `code`（机器码，用于分支判断）+ `message`（给用户看的提示）。
- 不要依赖 HTTP 状态码判断业务成败，一律看 `success`。

### 1.5 幂等键（Idempotency-Key）

所有写操作（尤其抽奖、兑换）必须在 Header 携带 `Idempotency-Key`：
- 客户端生成，建议格式：`lottery_draw_<timestamp>_<random>`。
- **重试时必须复用同一个 key**，防止重复扣积分/重复抽奖。
- 相同 key 重复请求返回首次结果（幂等回放）；同 key 不同参数返回 409 `IDEMPOTENCY_KEY_CONFLICT`。
- 缺失返回 400 `MISSING_IDEMPOTENCY_KEY`。
