## 🔍 快速查找指南

### 按问题类型查找

**API相关问题**
- API接口不匹配、404错误 → `01-核心开发质量标准.mdc`
- API响应格式不一致 → `08-生产质量保证标准.mdc` 第1章

**性能问题**
- 工具调用重复、效率低下 → `02-工具调用优化系统.mdc`
- N+1查询、Redis阻塞 → `08-生产质量保证标准.mdc` 第7章
- 慢查询、缺少索引 → `08-生产质量保证标准.mdc` 第7章

**服务部署问题**
- 进程冲突、端口占用 → `03-服务进程管理系统.mdc`
- 健康检查失效 → `08-生产质量保证标准.mdc` 第2章
- 配置环境变量缺失 → `08-生产质量保证标准.mdc` 第3章

**安全问题**
- WebSocket未鉴权 → `08-生产质量保证标准.mdc` 第4章
- 权限缓存失效 → `08-生产质量保证标准.mdc` 第8章
- 幂等性缺失 → `08-生产质量保证标准.mdc` 第6章

**数据一致性问题**
- 事务处理错误 → `01-核心开发质量标准.mdc` + `08-生产质量保证标准.mdc` 第6章
- 数据库迁移混乱 → `08-生产质量保证标准.mdc` 第5章
- 时间时区不一致 → `07-日期时间处理标准.mdc`

**开发工作流问题**
- 编码格式错误 → `06-编码Git管理规范.mdc`
- Git操作风险 → `06-编码Git管理规范.mdc`
- 问题诊断慢 → `04-问题诊断解决系统.mdc`
- 文档冗余膨胀 → `05-文档生成质量控制.mdc`

### 按开发阶段查找

**设计阶段**
1. 查阅 `01-核心开发质量标准.mdc` 了解API接口规范
2. 查阅 `07-日期时间处理标准.mdc` 确定时间处理方案
3. 查阅 `08-生产质量保证标准.mdc` 了解安全和性能要求

**开发阶段**
1. 遵循 `01-核心开发质量标准.mdc` API和数据库规范
2. 应用 `02-工具调用优化系统.mdc` 优化性能
3. 使用 `07-日期时间处理标准.mdc` 处理时间数据
4. 参考 `06-编码Git管理规范.mdc` 提交代码

**测试阶段**
1. 对照 `08-生产质量保证标准.mdc` 质量检查清单
2. 使用 `04-问题诊断解决系统.mdc` 快速定位问题
3. 验证 `03-服务进程管理系统.mdc` 服务稳定性

**部署阶段**
1. 执行 `08-生产质量保证标准.mdc` P0级检查
2. 配置 `03-服务进程管理系统.mdc` 进程管理
3. 验证 `08-生产质量保证标准.mdc` 健康检查

**运维阶段**
1. 监控 `03-服务进程管理系统.mdc` 服务健康
2. 分析 `02-工具调用优化系统.mdc` 性能指标
3. 应用 `04-问题诊断解决系统.mdc` 排查问题

## 🎯 优先级标准

### P0 级别（必须立即遵守）
- API响应格式统一（08-第1章）
- WebSocket身份验证（08-第4章）
- 幂等键唯一约束（08-第6章）
- 生产配置安全（08-第3章）
- 健康检查真实实现（08-第2章）

### P1 级别（应尽快遵守）
- 结构化日志规范（08-第2章）
- 配置自动校验（08-第3章）
- N+1查询优化（08-第7章）
- 权限缓存失效（08-第8章）
- 工具调用缓存（02-全章）

### P2 级别（技术债务改进）
- 慢查询优化（08-第7章）
- 事务边界清晰（08-第6章）
- 文档质量控制（05-全章）
- 进程管理优化（03-全章）

## 💡 使用建议

1. **首次使用**: 先阅读总则（00），再根据具体问题查阅对应章节
2. **日常开发**: 将常用规范（01/08）加入IDE书签
3. **代码审查**: 对照质量检查清单（08-第9章）逐项检查
4. **问题排查**: 使用快速诊断系统（04）定位问题
5. **知识沉淀**: 新问题及时反馈，更新规范文档
