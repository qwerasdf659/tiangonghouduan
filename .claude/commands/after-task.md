任务完成后执行完整的验证和清理流程。

## 第一步：项目运行验证

### 必须检查：
1. **启动项目**
   ```bash
   npm start
   ```
   - 是否正常启动？
   - 是否有报错？
   - 是否有警告？

2. **运行测试**
   ```bash
   npm test
   ```
   - 测试是否通过？
   - 新功能是否测试覆盖？
   - 原有测试是否仍然通过？

3. **代码检查**
   ```bash
   npm run lint
   ```
   - 是否有语法错误？
   - 是否有代码规范问题？

## 第二步：功能验证

### 检查清单：
- [ ] 新增功能是否正常工作？
- [ ] 原有功能是否受影响？
- [ ] 是否有破坏性改动？
- [ ] 边界条件是否处理？
- [ ] 错误处理是否完善？

## 第三步：临时文件清理

### 搜索并删除：
```bash
# 查找临时文件
find . -name "*.tmp" -o -name "*.bak" -o -name "*~"

# 查找测试临时文件
find . -name "*test*.temp.*" -o -name "*debug*"
```

### 清理清单：
- [ ] 删除临时测试文件
- [ ] 删除调试文件
- [ ] 删除备份文件
- [ ] 删除过时的配置文件

## 第四步：无用文件清理

### 检查重复和无用文件：
- [ ] 是否有重复的脚本？
- [ ] 是否有无用的 .md 文档？
- [ ] 是否有未使用的组件？
- [ ] 是否有冗余的工具函数？

### 删除标准：
- ✅ 功能重复 → 删除
- ✅ 不再使用 → 删除
- ✅ 已废弃 → 删除或移到 archived/
- ✅ 临时调试 → 删除

## 第五步：引用清理

### 检查并移除：
```bash
# 搜索对已删除文件的引用
grep -r "已删除的文件名" .
```

- [ ] 移除 import/require 语句
- [ ] 更新文档中的路径引用
- [ ] 更新配置文件中的引用
- [ ] 更新 README 中的说明

## 第六步：问题修复

### 如果发现问题：
1. **立即修复**
   - 不要留下破坏性改动
   - 确保项目处于可运行状态

2. **重新验证**
   - 修复后再次运行测试
   - 确认问题已解决

3. **记录修复**
   - 说明发现的问题
   - 说明修复方案

## 第七步：最终确认

### 完成检查清单：
- [ ] ✅ 项目可正常运行
- [ ] ✅ 所有测试通过
- [ ] ✅ 无语法错误
- [ ] ✅ 原有功能正常
- [ ] ✅ 新功能工作正常
- [ ] ✅ 临时文件已清理
- [ ] ✅ 无用文件已删除
- [ ] ✅ 无效引用已移除
- [ ] ✅ 代码注释完整
- [ ] ✅ 符合项目规范
