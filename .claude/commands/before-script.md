在生成任何脚本之前，执行以下强制检查流程：

## 第一步：全面搜索现有功能

检查以下目录是否已有类似功能：
- `/scripts/` - 主脚本目录
- `/scripts/archived/` - 已归档脚本
- `/scripts/database/` - 数据库相关
- `/scripts/maintenance/` - 维护脚本
- `/scripts/toolkit/` - 工具包
- `/utils/` - 工具函数

搜索命令示例：
```bash
# 搜索相关关键词
grep -r "关键功能词" scripts/
grep -r "相关业务逻辑" utils/
```

## 第二步：评估是否可以扩展现有功能

如果找到相关脚本，评估：
- 是否可以添加参数来扩展功能？
- 是否可以添加函数来增强功能？
- 是否可以重构现有代码来支持新需求？

## 第三步：新脚本必要性评估

只有同时满足以下条件才能创建新脚本：
- ✅ 现有脚本确实不包含相关功能
- ✅ 新功能与现有功能职责明确分离
- ✅ 新脚本有独特且不可合并的业务价值
- ✅ 不会增加技术债务
- ✅ 不会增加系统复杂度

## 第四步：创建脚本规范

如果确认需要创建新脚本：
- 文件命名清晰描述功能
- 添加详细的文件头注释（目的、用法、作者、时间）
- 使用北京时间
- 遵循现有脚本的代码风格
- 添加详细的中文注释

## 第五步：更新文档

创建新脚本后必须：
- 更新 `scripts/README.md`
- 说明脚本用途和使用方法
- 说明与其他脚本的关系
